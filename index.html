Dependency Inversion Principle

High-level modules should not depend on low-level modules.
Both should depend on abstractions.

Layers

Separate and organize code by it's role/responsibility in the system.
  • Presentation
    Views, View Models, Controllers
  • Domain
    Entities, Domain Services, Domain Events
  • Persistence
    Repositories, ORM

                          In order to keep track of the benchmark
                          As a product manager
                          I need to know if offers will be rated too high/low

                          Scenario: GoldCar is rated too high
                            Given The the benchmark-algorithm rates an offer of "GoldCar"
                            And The benchmark is at least "7.0" (Good)
                            Then A warning-email should be sent to "productmanagement@billiger-mietwagen.de"

SwiftMailerAdapter


                        namespace Infrastructure;

                        use Domain\Mailer;

                        class SwiftMailerAdapter implements Mailer
                        {
                            public function __construct(\Swift_Mailer $mailer)
                            {
                                $this->mailer = $mailer;
                            }

                            public function send($recipient, $message)
                            {
                                $swiftMessage = new \Swift_Message();
                                $message->setTo($recipient);
                                $message->setBody($message);

                                $this->mailer->send($swiftMessage);
                            }
                        }
